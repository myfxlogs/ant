# 市场指标体验与自定义指标资产设计

本文档在已落地的「内置指标 + 市场状态检测」能力之上，定义 **前端体验重构（A+B+C）**、**图表化展示路径**、**多语言下的 AI 需求输入**，以及 **AI 生成自定义指标作为用户私有数字资产**（保存、删除、共享、未来市场售卖/租赁）的产品与技术方案。

**与既有文档的关系：**

- 规则模型、参数扫描、`DetectMarketRegime`、`market_regimes.features` 等仍以 **`docs/专项设计/未完成项/市场状态模型训练与集成专项设计.md`** 为纲；
- 本文档专注 **人机交互、可视化、自定义指标生命周期与商业化预留**，不重复该文档中已实现的规则与表结构细节；
- 接口形态须符合 **`docs/接口与数据流架构约定.md`**：新增能力以 **ConnectRPC / gRPC / SSE** 为面，禁止新增业务 REST 作为主 API。

---

## 1. 背景与问题

### 1.1 现状

- 「内置指标管理」页（`/strategy/market-regime/indicators`）以 **账号/品种必选 + 表格展示标量** 为主，**缺少 K 线/副图**，与用户对 **MetaTrader 类客户端「看图识指标」** 的心智差距大。
- 当前 19 个内置指标在服务端为 **整段 K 线末端标量**，写入 `features`，**非逐 bar 时间序列**，难以直接绑 MT 式副图曲线（需扩展，见 §4）。
- 「AI 写指标」在简化方案中退化为 **从内置库推荐**；用户原始诉求仍是 **自然语言 → 可运行/可保存的自定义指标**，与资产化、商业化需求尚未对齐。

### 1.2 目标

1. **体验**：在账号/品种仍须用户选择的前提下，落实 **A+B+C**（一键解读、嵌入高频路径、AI 优先），并分阶段引入 **主图 K 线 + 副图指标曲线**。
2. **语言**：AI 需求输入 **不限中文**，与用户选择的 **界面语言 / 输入语言** 一致即可；系统与模型应 **多语言理解**，不假设语种。
3. **资产**：用户通过 AI 得到的 **自定义指标** 视为 **用户私有财产**，支持 **保存、删除、共享**，并预留 **市场售卖、租赁** 能力（合规与计费另文细化时可引用本文档资产模型）。

### 1.3 非目标（第一阶段可不实现）

- 不在本文档阶段强制实现 **完整 MT 级 EA 编辑器** 或 **tick 级高频自定义指标**（安全与资源边界仍遵循市场状态专项设计中的禁止项）。
- 市场 **撮合、支付、税务** 等不在本文展开，仅以「能力预留 + 数据模型钩子」描述。

## 2. 信息架构与页面流程（A+B+C，账号/品种必选）

### 2.1 单页主路径（内置环境解读台）

1. **上下文（必选）**：账号、品种（支持搜索过滤）、周期、K 线条数。
2. **主行动**：单一主按钮 **「分析当前环境」**（或等价文案），一次触发检测并刷新解读区。
3. **解读区（默认折叠技术细节）**：
   - **一句话结论**（规则模板为主，可选 LLM 润色，输出语言与 §3 一致）；
   - **市场状态**（regime + 置信度）；
   - **3～5 个与当前状态强相关的关键指标数值**（由规则映射，不必默认展示全部 19 项）；
   - **图表区**（§4）：至少主图 K 线；副图分阶段叠加。
4. **AI 区**：自然语言描述需求 → **不限语种** → 返回推荐内置指标 key 列表 + 理由；并与「副图显示哪些序列」联动（见 §4.3）。
5. **进阶**：**「展开全部 19 项技术指标」** 表格，与当前实现兼容，供进阶用户查看。

### 2.2 嵌入（B）

- **市场状态检测页**（`/strategy/market-regime`）：检测完成后复用同一 **解读 + 图表** 组件，避免仅展示原始 JSON。
- **调度健康**：在已有 regime/漂移提示旁，增加 **「环境快照」** 入口或摘要组件（只读），链回完整解读台（可选）。

## 3. 多语言与 AI 需求输入

### 3.1 原则

- **用户用何种自然语言描述需求，由用户决定**，与前端 **i18n 所选界面语言** 可以一致也可以不一致（例如界面英文、需求写中文）。
- **不要求用户使用中文**；产品文案、占位符、按钮提示随界面语言切换。
- **AI 服务**（`DescribeMarketFeatures` 或后续流式/对话接口）应：
  - 接受 **UTF-8 任意自然语言** 输入；
  - 在 **system prompt** 中明确「根据用户输入语言，用**同一语言**生成摘要与理由」；若输入为混合语言，以 **用户输入主体语种** 为准。

### 3.2 前端实现要点

- 需求输入框 **不**写死中文 placeholder；使用 i18n key（如 `marketRegime.indicators.aiPlaceholder`）各语言一套。
- 向服务端传 **原始字符串** 即可，**无需**额外传 `locale` 亦可工作；若需提高稳定性，可 **可选** 附加 `ui_locale`（BCP 47，如 `zh-CN` / `en`）供服务端记录审计与 A/B，**不作为强制字段**。

### 3.3 服务端实现要点

- 提示词中禁止出现「请用中文回答」等硬编码；改为「Match the user's language」或等价英文规则，避免英文用户得到中文摘要。

## 4. 图表与数据：分阶段对齐 MT 观感

### 4.1 问题根因

当前 `computeBuiltinIndicators` 对整段 `klines` 输出 **各指标一个标量**，适合写入 `market_regimes.features` 与调度漂移；**不足以**绑 MT 式「每根 K 一个点」的副图曲线。

### 4.2 阶段 1（优先落地、改动相对小）

- **主图**：使用前端已有依赖（如 `lightweight-charts`）绘制 **OHLC K 线**；K 线数据与 `DetectMarketRegime` **同源**（推荐由同次 RPC 返回 `candles[]` 或等价结构，避免二次拉取不一致；若短期无法改 proto，则约定并行只读 K 线接口且时间范围严格一致，并在文档中记为**临时方案**）。
- **副图 0～1**：可先上 **成交量** 或与 close 强相关、且可在前端由 K 线安全重算的简单序列（若与内置公式不完全一致，**必须在 UI 标注「示意」**）。

### 4.3 阶段 2（与 19 内置指标名实一致）

- 后端增加 **按 bar 滑动窗口** 的指标序列计算（或对每根 bar 输出末值），通过 **ConnectRPC** 返回，例如：
  - 扩展 `DetectMarketRegimeResponse` 增加 `indicator_series`；或
  - 新增 `GetBuiltinIndicatorSeries`（参数：account、symbol、timeframe、count、keys[]）。
- 前端 **主图 + 1～3 个可切换副图**（`lightweight-charts` 多 pane 或 `recharts` 小多图），**AI 推荐 / 用户勾选** 决定默认展示的 `keys[]`。

### 4.4 与 AI 的联动

- AI 返回的指标 key → 自动加入「副图候选」列表；用户可取消勾选。
- 流式输出（若已实现前端打字机）可保留；**真正 token 级流式**若走独立 Connect stream，须单独在 `docs/接口与数据流架构约定.md` 中登记，**禁止**用轮询冒充流式。

## 5. 自定义指标（AI 生成）作为用户私有资产

### 5.1 定义

- **自定义指标**：在用户主导下，由 **AI 生成或辅助生成** 的、可版本化的指标定义（可为 **DSL**、**受限 Python**、或 **声明式配置 + 内置算子组合** 等形式，具体实现阶段选型），经 **沙箱校验 / 人工确认**（沿用市场状态专项设计中安全边界）后，绑定到用户账户。
- 指标及其产出序列的 **知识产权与使用权默认归属创建用户**，平台提供托管与运行环境。

### 5.2 用户权利（产品必选能力）

| 能力 | 说明 |
|------|------|
| **保存** | 持久化名称、版本、描述、源码或 DSL、校验报告摘要、可见性标记。 |
| **删除** | 软删除或硬删除策略由实现阶段定；删除后停止运行与共享链接。 |
| **共享** | 生成只读或可 fork 的 **共享链接/组织内共享**；权限粒度：仅查看定义 / 可复制为自有副本。 |
| **售卖 / 租赁（预留）** | 上架到 **指标市场**（名称占位）：定价模式（一次性、订阅、按调用次数）、分成比例、下架、审计日志；**支付与合约**另文或与商务/合规专项对接。 |

### 5.3 数据模型（草案，实施时落到迁移与 proto）

建议独立于 `market_regime_models` 的 **用户指标资产表**（名称示例）：

- `user_custom_indicators`：`id`, `owner_user_id`, `name`, `slug`, `definition`（加密或对象存储引用）, `status`（draft / validating / active / archived）, `visibility`（private / link / org / marketplace）, `license`（none / sale / rent）, `price_json`, `created_at`, `updated_at` 等。
- `user_custom_indicator_versions`：版本、校验报告、发布说明。
- **审计**：创建、发布、共享变更、上架、下架、交易相关操作写入审计表。

与 **`市场状态模型训练与集成专项设计.md`** 中已延后的 `market_regime_indicators` 等表：**可合并选型或拆表**，以「用户资产」为第一建模目标，避免与「系统内置 19 项」概念混淆。

### 5.4 运行与安全（与既有禁止项对齐）

- 自定义指标 **不得** 直接下单或改调度；仅产出 **数值/标签/告警建议**。
- **长驻运行**须在策略服务或隔离 Worker 中执行，**禁止**在 API 进程内执行用户代码（与既有设计一致）。
- 上架市场前须 **静态检查 + 沙箱回放 + 资源限额** 通过（细则可引用原专项附录中的延后项并收敛为实施清单）。

### 5.5 接口形态（草案）

- ConnectRPC：`CreateUserCustomIndicatorDraft`、`ValidateUserCustomIndicator`、`PublishUserCustomIndicator`、`DeleteUserCustomIndicator`、`ShareUserCustomIndicator`、`ListUserCustomIndicators`、市场相关的 `ListMarketplaceIndicators`（预留）等。
- **禁止**以 REST 作为上述能力的主面对外接口。

## 6. 实施阶段建议

| 阶段 | 内容 | 依赖 |
|------|------|------|
| **P0** | 内置页信息架构调整（主按钮、默认四块、19 项折叠）；i18n 占位与 AI 多语言提示词；市场状态页嵌入同一组件 | 前端为主 |
| **P1** | K 线主图 + 合规的数据来源（proto 扩展或临时双请求 + 文档记载例外） | 前端 + 轻量后端 |
| **P2** | 内置指标时间序列 API + 副图与 AI 勾选联动 | 后端计算扩展 |
| **P3** | 用户自定义指标：表结构、草稿、校验、保存删除共享 | 沙箱与策略服务 |
| **P4** | 指标市场：上架、租赁、计费、分成 | 支付、合规、运营 |

**P3/P4** 须在更新 **`docs/接口与数据流架构约定.md`** 后再动接口与库表。

## 7. 验收要点（本文档范围）

- 用户在 **任意支持语言** 下输入需求，AI 摘要与理由 **语种匹配** 用户输入（或明确的中性 fallback 策略且经产品确认）。
- 「分析当前环境」路径下，默认展示 **非表格优先**；进阶表格可展开。
- 阶段 1 至少 **主图 K 线** 可用；阶段 2 至少 **1 个内置指标** 与 K 线时间对齐的副图曲线。
- 自定义指标保存后 **仅 owner 可写**；共享与上架有 **权限与审计**；删除后不可再被他人引用（引用方展示「已失效」）。
