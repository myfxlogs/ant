## 一、NautilusTrader ⭐⭐⭐⭐⭐ — 综合评分最高

**仓库**：[nautechsystems/nautilus_trader](https://github.com/nautechsystems/nautilus_trader)（10k+ stars，澳洲机构主导）
**栈**：Rust core + Cython + Python（高性能 + 易用）
**license**：LGPL v3（架构思想可借鉴，**代码直接复制需评估 license 兼容性**）

### 对 ant 最值得学的 4 件事

| ant 卡片 | Nautilus 对应模块 | 具体代码路径 |
|---|---|---|
| **M10-BASE-B1** OMS 15-state 状态机 | `ExecutionEngine` + 完整 `OrderStatus` 枚举 + `OrderEventHandler` | `nautilus_trader/execution/engine.py`<br>`nautilus_core/model/src/orders/` |
| **M10-BASE-B4 / B5 / B6** event-sourced ledger | `MessageBus` 模式 + `Cache` 全状态可重建 | `nautilus_core/common/src/msgbus.rs`<br>`nautilus_trader/cache/cache.py` |
| **M10-BASE-C6** Sizer 范式（Signal→HardLimit→Sizer→Submit） | `RiskEngine` 是流水线节点（不是二值 deny），订单可被改写 | `nautilus_trader/risk/engine.py` |
| **M10-BASE-D4 / D5** FillModel 拆分 + 回测悲观化 | `BacktestEngine` 强制 `FillModel` + `LatencyModel`，**不允许 0 延迟 0 滑点** | `nautilus_trader/backtest/models.py`<br>`nautilus_trader/backtest/engine.py` |

### Nautilus 做对而 ant 做错的关键

1. **同一段代码、两种 source**——`LiveDataEngine` 和 `BacktestDataEngine` 共用 `DataEngine` 基类，**但回测强制注入滑点和延迟模型**。
   - 比 ant `spec/21` 的"完全相同代码路径"理念**安全得多**（ant 现在违反 M11-7 §三·G）

2. **Actor model**：每个 Strategy 是独立 actor，通过 MessageBus 通信
   - **天然支持 event sourcing 和重放**——这正是 M10-BASE-B4 想做的
   - NATS JetStream 升格为 Tier-0 时，**event schema 和命名应直接对齐 Nautilus 的 `OrderEvent` / `PositionEvent` / `AccountState` 类型层级**——这能让未来如果要把 ant 策略层迁到 Nautilus（或反过来）成本最低

3. **OrderStatus 完整枚举**：包含 `REQUOTED` / `EXPIRED` / `TRIGGERED` / `REJECTED` / `CANCELED` / `PENDING_UPDATE` / `PENDING_CANCEL` 等 ant 缺失或概念混淆的状态
   - 对应 M10-BASE-B1 的 15 个状态——可直接抄状态命名和转换图

### 学习方式

**不是 fork 全部**（Rust+Cython 学习曲线陡、license 不兼容），而是**只抄设计**：
- 阅读 `msgbus.rs`（~300 行 Rust）→ 翻译成 Go 实现到 ant Tier-0 ledger
- 阅读 `risk/engine.py`（~400 行 Python）→ 翻译到 `risksvc/pipeline.go`
- 阅读 `backtest/models.py`（~200 行）→ 翻译到 `oms/fill_model.go`

**总阅读量 ≈ 1000 行核心代码 ≈ 1 个工作日**，但能给 M10-BASE Phase B + C + D 提供完整设计模板。

## 二、QuantConnect Lean ⭐⭐⭐⭐⭐ — 架构教科书

**仓库**：[QuantConnect/Lean](https://github.com/QuantConnect/Lean)（10k+ stars，C#/.NET，企业级）
**license**：Apache 2.0（**license 友好**，代码可借鉴）
**对 ant 最重要的贡献**：**五段算法框架（Algorithm Framework）**

### 五段架构（直接对应 M10-BASE Phase C 设计目标）

```
Universe Selection  →  Alpha Generation  →  Portfolio Construction  →  Risk Management  →  Execution
   (要交易什么)         (信号方向 + 强度)       (信号 → 目标权重)            (软调节)             (订单生成)
```

**这正是 ant 的风控范式应该改成的样子**（M10-BASE-C6 的精确模板）：

| Lean 模块 | 设计本质 | ant 对应卡片 |
|---|---|---|
| **Alpha Model** | 输出 `Insight{Direction, Magnitude, Confidence, Period}`，**不是裸订单** | M10-BASE-C6（quantengine 输出应升级为 Insight） |
| **Portfolio Construction Model** | 把 N 个 Insight → 目标权重（含 Black-Litterman / risk-parity 实现） | M10-BASE-C2/C3（Sizer 接口完整设计） |
| **Risk Management Model** | **目标权重的修改器**，不是二值 deny（这就是 "Sizer 范式"） | M10-BASE-C1/C6（HardLimit + Sizer 流水线） |
| **Execution Model** | 决定怎么从当前权重达到目标权重（TWAP/VWAP/IS） | **M11-13** 直接对应 |

### 具体可抄的代码（Apache 2.0 友好）

| 文件 | 学什么 | 对应卡片 |
|---|---|---|
| `Algorithm.Framework/Risk/MaximumDrawdownPercentPortfolio.cs` | drawdown-based scaling 标准实现 | M10-BASE-C2 |
| `Algorithm.Framework/Portfolio/MeanVarianceOptimizationPortfolioConstructionModel.cs` | Markowitz 标准实现 | M10-BASE-C2/C3 |
| `Algorithm.Framework/Portfolio/BlackLittermanOptimizationPortfolioConstructionModel.cs` | BL 优化 | M10-BASE-C3 进阶 |
| `Algorithm.Framework/Execution/VolumeWeightedAveragePriceExecutionModel.cs` | VWAP 标准实现 | M11-13 |
| `Algorithm.Framework/Execution/StandardDeviationExecutionModel.cs` | σ-based execution timing | M11-13 |
| `Engine/DataFeeds/Enumerators/RealTimeProviderConsolidator.cs` | bar 边界处理（含 timezone） | M10-BASE-F2 SessionClock |
| `Common/Securities/SymbolProperties.cs` | symbol metadata（lot size / tick size / margin） | ant 现有 SymbolParams 升级参考 |

### 唯一缺点

C#/.NET 生态对 ant（Go+Python）有翻译成本。但**架构理念是普世的**，且 Lean 代码注释极其充分，**阅读门槛低于 Nautilus 的 Rust core**。

### 推荐学习路径

1. 先读 `Algorithm.Framework/README.md`（10 min）—— 五段框架概览
2. 再读 `Algorithm.Framework/Risk/` 目录的所有 .cs（~600 行，1h）—— 设计每个 Risk Model 的范式
3. 最后读 `Algorithm.Framework/Portfolio/MeanVarianceOptimization*.cs`（~400 行，1h）—— 看完整的 Markowitz + 约束求解
4. 输出：把 `docs/spec/23-risk-management.md` 重写为 **Alpha → Portfolio → Risk → Execution** 四段，**直接对标 Lean 的 `IAlphaModel` / `IPortfolioConstructionModel` / `IRiskManagementModel` / `IExecutionModel` 四个接口**

**license**：MIT
**对 ant 的价值**（如果未来要接中国市场 broker）

CTP / CTPMini / QMT / 迅投 / IB / Binance / OKX...

### 2. EventEngine（简化版 MessageBus）

和 NautilusTrader 同思路但简单 10 倍，**Go 翻译成本最低**。

### 注意

**不推荐**直接抄整体架构——event sourcing / async 模型都不如 Nautilus。
