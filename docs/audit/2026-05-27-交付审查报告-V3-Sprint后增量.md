### V3-R-8 Python sandbox 超时是 **threading + ctypes 弱实现**，非计划要求的 multiprocessing

`@/opt/ant/strategy-service/app/engine/sandbox.py:300-330`：用 `threading.Timer + ctypes.pythonapi.PyThreadState_SetAsyncExc` 注入异步异常。

**已知问题**：
- C 扩展中（numpy / pandas 大量内部循环）async exc **被忽略**
- `while True: pass` 极端情况下 GIL 不让步，超时可能延迟数秒
- 卡在 `socket.recv` / `time.sleep(99999)` 等 C 阻塞调用时**完全无效**

计划 §6 §S0.6 / §E-6 明确要求 `multiprocessing + Process.terminate`，AI 选择了较弱实现且 commit message 自称 "enforce multiprocessing timeout"（**虚假声明**）。

**风险**：恶意/低质策略可以 DoS strategy-service 的 worker pool；当前 `BACKTEST_SEMAPHORE=2`，2 个恶意请求即可让整个回测服务挂死。

**借鉴**：spread/inventory 计算在 hummingbot `pure_market_making.py` 中是策略的一部分；AntTrader 可把 costsvc 暴露成策略可用的 Python helper（`ctx.cost.estimate(...)`），让 AI 生成的策略能感知成本而非"只看价格信号"。

### 4.5 ccxt-pro（多语言，MIT）— 33k stars

主要是协议层，不可比策略层。但**AntTrader 的 mt4/mt5 适配器**目前是单一 `mtapi.io` 服务的封装；ccxt-pro 思路是 trait + multi-impl。如果未来想加 IBKR / OANDA / Binance，当前架构需要重写 adapter 层（mdgateway 现在硬绑 mtapi）。
