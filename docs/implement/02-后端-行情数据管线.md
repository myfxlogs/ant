## 2. 管线架构总览

```
mtapi.OnQuote
  → adapter/mt[45]/gateway.go（proto → Tick DTO）
  → Manager.HandleTick(tick)
      ├─ 0. StuffingDetector.IsPaused     → 报价填充预检
      ├─ 1. Normalizer.Resolve            → (broker, raw) → canonical  [OTel span]
      ├─ 2. Quality.Check                 → bid>ask / non-positive / clock-skew 丢弃 [OTel span]
      ├─ 3. StuffingDetector.Observe      → 报价填充后检
      ├─ 4. Spread 异常检测               → Z-score > 3.0
      ├─ 5. TickDedup.Seen                → 1000 条窗口去重 [OTel span]
      ├─ 6. MarketState.Update            → 市场状态（M10-BASE-F1）
      ├─ 7. BarAggregator.AddTick         → 6 周期 OHLCV [OTel span]
      ├─ 8. Publisher (Tick + Bar)        → NATS JetStream [OTel span]
      └─ 9. CHWriter.Enqueue (Tick + Bar) → 异步批量写 CH [OTel span]
```

## 3. Runner（runner.go）

`Run(ctx, deps)` 是管线装配入口，完整启动序列：

```go
1. 初始化 OTel Tracer
2. SpillWriter 创建（DefaultSpillConfig）
3. CHWriter 创建（DefaultCHWriterConfig）
4. Publisher 创建（NATS JetStream）
5. SpillReplay.Run()  → 重放未提交 spill（双写 NATS + CH）
6. aggregator.LoadFinalizedBars(ch)  → 从 CH 加载已终态 bar
7. go chWriter.Start(ctx)  → 后台 flush goroutine
8. Backfiller 启动（初始扫描 + 6h cron）
9. NormalizerInvalidator 启动（PG LISTEN + ticker fallback）
10. Manager 创建
11. chWriter.SetOnSpillFail(func(brokerKey, err) { log.Warn })
12. 从 PG 加载所有 active account → 逐账户启动 Gateway
    for each account:
      a. mt4.New(cfg) 或 mt5.New(cfg)
      b. gw.Connect(ctx) → mtapi Connect
      c. gw.FetchBrokerInfo(ctx) → OnBrokerInfo
      d. mgr.AddGateway(ctx, gw)
      e. hub.Register(accountID, Session, executor)
      f. gw.Subscribe(ctx, symbols, mgr.HandleTick)
      g. gw.SubscribeProfit(ctx, OnAccountProfit)
      h. gw.SubscribeOrderUpdate(ctx, OnOrderUpdate)
13. go healthMonitor(ctx, mgr, chWriter, log, OnAccountDisconnect)
14. startAccountEventSubscriber(ctx, natsConn, log)
15. <-ctx.Done() → drain + flush
```

**健康监控（healthMonitor）**：每 30s 检查：

| 状态 | 条件 | 动作 |
|------|------|------|
| `no_data` | 从未收到 tick | debug |
| `healthy` | 最近 5min 内有 tick | — |
| `stale` | 5-15min 无 tick | warn |
| `dead` | >15min 无 tick | error + onDisconnect + syncHistory |

**S-2 内存保护**：每 30s 检查 cgroup 内存（v1 + v2 兼容）：

```
memRatio > 80% → chWriter.SetBufferEnabled(false)  // 绕过 CH Buffer
memRatio < 60% → chWriter.SetBufferEnabled(true)   // 重新启用
```

### HandleTick 管线

`HandleTick(t *Tick)` 精确的 10 阶段处理（含 6 个 OTel span）：

```go
// 阶段 0：报价填充预检
if stuffingDetector.IsPaused(broker, canonical) { return }

// 阶段 1：规范化 [OTel span]
t.Canonical = normalizer.Resolve(ctx, broker, SymbolRaw)

// 阶段 2：质量检查 [OTel span]
qr := quality.Check(ctx, t)
if qr.Dropped { return }

// 阶段 3：报价填充后检
if stuffingDetector.Observe(broker, canonical) { return }

// 阶段 4：Spread 异常
if qr.SpreadBps > 0 {
    z := quality.SpreadZscore(key, spreadBps)
    if z > MaxSpreadZscore { RecordSpreadAnomaly() }
}

// 阶段 5：去重 [OTel span]
if dedup.Seen(t) { return }

// 阶段 6：更新 lastTickAt

// 阶段 7：市场状态
if marketState != nil { marketState.Update(t) }

// 阶段 8：Bar 聚合 [OTel span]
aggregator.AddTick(t, onBar)

// 阶段 9：NATS 发布 [OTel span]
publisher.PublishTick(ctx, t)
for bar := range bars { publisher.PublishBar(ctx, bar) }

// 阶段 10：CH 入队 [OTel span]
chWriter.EnqueueTick(t)
for bar := range bars { chWriter.EnqueueBar(bar) }
```

## 5. Normalizer（normalizer.go）

```go
func (n *Normalizer) Resolve(ctx, broker, raw string) string {
    // 1. 缓存查询（map, max 100k 条，超限重置）
    // 2. PG broker_symbols 表查询（3s timeout）
    // 3. 算法 fallback: stripSuffix(raw)
}

func stripSuffix(raw string) string {
    // 去除 .m / .pro / .x / .c 后缀（点号后）
    // 去除 _i / _r / _institutional / _retail 后缀
    // 全部转大写
    return strings.ToUpper(s)
}
```

缓存失效通过 `NormalizerInvalidator`：监听 PG NOTIFY `broker_symbols_changed` 或 30s ticker。

## 6. Quality（quality.go）

### 配置

```go
type QualityConfig struct {
    GapMaxSeconds    float64 // 默认 5
    OutlierSigma     float64 // 默认 5
    SkewMaxSeconds   float64 // 默认 30
    MaxClockSkewMs   int64   // 默认 5000
    HistorySize      int     // 默认 100
    MaxSpreadZscore  float64 // 默认 3.0（M10-BASE-F5）
    MaxTickRateZscore float64 // 默认 4.0（M10-BASE-F4）
}
```

### 硬丢弃条件

| 条件 | Reason | 指标 |
|------|--------|------|
| `Bid > Ask` | `bid_gt_ask` | `md_tick_dropped_total{reason="bid_gt_ask"}` |
| `Bid <= 0 \|\| Ask <= 0` | `non_positive` | `md_tick_dropped_total{reason="non_positive"}` |
| `\|ArrivedUnixMs - TsUnixMs\| > 5000` | `clock_skew` | `md_tick_dropped_total{reason="clock_skew"}` |

### 统计但不丢

| 检查 | 方法 | 阈值 |
|------|------|------|
| 离群值 | MAD (Median Absolute Deviation) | 5σ |
| Tick 间隔 | 与上次到达时间的差值 | 5s |
| Spread 异常 | Z-score 历史分布 | 3.0 |
| 时钟偏差 | ArrivedUnixMs - TsUnixMs | 5000ms |

可选 DLQWriter 用于丢弃 tick 的采样分析（M10 ADR-0010 §2.2）。

## 10. CHWriter（clickhouse_writer.go）

```go
type CHWriterConfig struct {
    FlushInterval time.Duration // 默认 500ms（M10 从 1s 降到 500ms）
    MaxBatchSize  int           // 默认 10000（M10 从 1000 升到 10000）
    QueueSize     int           // 默认 50000（M10 从 5000 升到 50000）
}
```

### 写入流程

```
EnqueueTick(tick) / EnqueueBar(bar)
  → 用户限流检查（nil-safe）
  → channel send（tickQ / barQ）
  → channel 满 → SpillWriter 落盘

Start(ctx):
  → ticker 500ms 触发 flush
  → batch >= 10000 触发 flush
  → ctx.Done 时 drain
```

### 目标表切换

```go
bufferEnabled=true  → md_ticks_buffer / md_bars_buffer（默认，CH Buffer engine）
bufferEnabled=false → md_ticks / md_bars（直写，绕过 Buffer engine）
```

运行时通过 `SetBufferEnabled(bool)` 动态切换（S-2 OOM 防护）。
