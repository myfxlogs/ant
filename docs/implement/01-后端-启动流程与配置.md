## 1. 主入口启动序列

`main()` 函数严格按依赖链顺序执行：


## 2. syncAccountHistory 闭包

定义在 `main()` 内部，用于账户断开后同步历史订单到 PG `trade_records`：

```go
syncAccountHistory := func(accountID string) {
    uid, _ := uuid.Parse(accountID)
    ctx, cancel := context.WithTimeout(ctx, 30s)
    defer cancel()

    from := time.Now().AddDate(-1, 0, 0)          // 默认拉 1 年
    if t, err := tradeRecordRepo.GetLastSyncTime(ctx, uid); err == nil && t != nil {
        from = *t                                   // 增量同步
    }
    to := time.Now()

    records, err := mthubSvc.OrderHistory(ctx, accountID, from, to)
    // 转换为 model.TradeRecord（含 AccountID, Ticket, Symbol, OrderType,
    //   Volume, OpenPrice, ClosePrice, Profit, Swap, Commission,
    //   OpenTime, CloseTime, OrderComment, MagicNumber, Platform）
    tradeRecordRepo.BatchCreate(ctx, tradeRecs)
}
```

**触发时机**：`OnAccountDisconnect` 和 `OnBrokerInfo` 回调。

## 3. 3 级保证金催缴检测（B-2.3）

```go
// Level 1 (预警): margin_level <= callPct * 1.5 → SSE only
// Level 2 (警告): margin_level <= callPct → SSE + Email
// Level 3 (危急): margin_level <= callPct * 0.7 → SSE + Email (1min cooldown)
```

保证金阈值从 PG `mt_accounts.broker_margin_call_pct` 加载（默认 100.0）。

**checkMarginCall 函数**：

1. 确定当前 level（基于 marginLevel 对比 callPct 的百分比）
2. 检查该 level 的上次发送时间：Level 1/2 冷却 5min，Level 3 冷却 1min
3. 未过冷却 → 发布 `TradeEventOrderMarginCall` 到 NATS JetStream
4. Level ≥ 2 → 同时调用 `emailNotifier.MarginCallAlert()`

## 4. OnAccountProfit 回调

mtapi `OnOrderProfit` 触发 → 同步 3 个动作：

1. **PG 更新**：
   ```sql
   UPDATE mt_accounts SET balance=$1, equity=$2, credit=$3, margin=$4,
       free_margin=$5, margin_level=$6, account_status='connected',
       updated_at=CURRENT_TIMESTAMP WHERE id=$7::uuid
   ```
2. **SSE 推送**：`mthubSvc.PublishAccountProfit(&AccountProfitEvent{...})`
3. **保证金检查**：`checkMarginCall()`

---

## 5. OnOrderUpdate 回调

mtapi `OnOrderUpdate` 触发 → 5 个动作：

1. **PG 更新**（同上）
2. **利润事件推送**：`accountBroker.Publish(AccountProfitEvent{含全量 Positions})`
3. **持仓快照推送**：`snapshotBroker.Publish(PositionSnapshot{含全量 PositionSnapshotItem})`
4. **风控聚合器更新**：`platformAgg.UpdatePosition()` + `platformAgg.ClearAccount()`
5. **平仓自动写入**：当 `o.UpdateType == "close"` 时，写入 `model.TradeRecord`

---

## 6. OnBrokerInfo 回调

连接成功后获取 broker margin call / stop out 比例并持久化：

```sql
UPDATE mt_accounts SET broker_margin_call_pct=$1, broker_stop_out_pct=$2
WHERE id=$3
```

零值（proto 未暴露）时保留 schema DEFAULT。

## 14. Server 启动（server.go）

```go
func Run(ctx, mux *ServeMux, port string, log *Logger) error {
    srv := &http.Server{
        Addr:         ":" + port,
        Handler:      mux,
        ReadTimeout:  15 * time.Second,
        WriteTimeout: 0,       // SSE/Stream 需要无限 write timeout
        IdleTimeout:  120 * time.Second,
    }
    // goroutine: ListenAndServe
    // select: 等待 ctx.Done() 或 listen error
    // Shutdown(10s timeout) 优雅关闭
}
```

**关键**：`WriteTimeout = 0` — 因为 SSE 和 ConnectRPC server-stream 需要长连接，不能有 write 超时。
