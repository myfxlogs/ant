# 00 · 技术栈与项目结构

## 技术栈

| 层次 | 技术 | 代码位置 |
|------|------|----------|
| 后端核心 | Go 1.24, ConnectRPC, pgx v5, ClickHouse-go v2, NATS, go-redis | `backend/cmd/server/main.go`, `backend/internal/` |
| 前端 | React 18, TypeScript, ConnectRPC-web, Zustand, TanStack Query, Ant Design, dayjs, i18next, Tailwind CSS | `frontend/src/` |
| 策略引擎 | Python 3.12, FastAPI | `strategy-service/` |
| 协议层 | Protobuf (Buf), ConnectRPC | `proto/ant/v1/` |
| 时序数据库 | ClickHouse 24 | `deploy/clickhouse/` |
| 消息队列 | NATS JetStream | `deploy/nats/` |
| 缓存/分布式锁 | Redis 8 | Docker Compose |
| 关系数据库 | PostgreSQL 18 | `backend/migrations/` |
| 监控 | Prometheus + Alertmanager | `deploy/prometheus/` |
| 容器编排 | Docker Compose | `docker-compose.yml` |

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 部署模型 | 单进程（ant-server） | 单机 docker-compose，进程内通信 < 1µs |
| 价格类型 | `decimal.Decimal`（shopspring） | 禁 float64，PG `NUMERIC(20,8)` / CH `Decimal(18,6)` |
| 时间 | UTC, 毫秒精度（int64） | 全系统统一 |
| RPC 框架 | ConnectRPC | 支持 HTTP/1.1 + HTTP/2，gRPC-web 兼容 |
| 行情消息 | NATS JetStream（fan-out） | 持久化 + 去重 + 可回溯 |
| 时序存储 | ClickHouse Buffer engine | 高写入吞吐，异步 flush |
| 前端状态 | Zustand（无 Redux） | 轻量、TS 友好、支持 persist |
| 实时推送 | ConnectRPC Server-Stream + SSE fallback | 统一 RPC 层，SSE 用于第三方集成 |
