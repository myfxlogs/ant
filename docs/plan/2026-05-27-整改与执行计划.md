### S0.1 给所有 admin RPC 加管理员拦截器

**问题**：V2 §1.S-1。`backend/internal/connect/admin/*` 7 handler 无管理员校验，任何登录用户可调。

**修复**：
1. 新建 `backend/internal/interceptor/admin.go`：
```go
package interceptor

import (
    "context"
    "connectrpc.com/connect"
    "github.com/google/uuid"
    "go.uber.org/zap"
)

type AdminChecker interface {
    IsAdmin(ctx context.Context, userID uuid.UUID) (bool, error)
}

type AdminInterceptor struct {
    checker AdminChecker
    log     *zap.Logger
}

func NewAdminInterceptor(c AdminChecker, log *zap.Logger) *AdminInterceptor {
    return &AdminInterceptor{checker: c, log: log}
}

func (i *AdminInterceptor) WrapUnary(next connect.UnaryFunc) connect.UnaryFunc {
    return func(ctx context.Context, req connect.AnyRequest) (connect.AnyResponse, error) {
        uid, err := uuid.Parse(GetUserID(ctx))
        if err != nil {
            return nil, connect.NewError(connect.CodeUnauthenticated, err)
        }
        ok, err := i.checker.IsAdmin(ctx, uid)
        if err != nil || !ok {
            i.log.Warn("admin denied", zap.String("user", uid.String()), zap.String("rpc", req.Spec().Procedure))
            return nil, connect.NewError(connect.CodePermissionDenied, fmt.Errorf("admin required"))
        }
        return next(ctx, req)
    }
}

func (i *AdminInterceptor) WrapStreamingClient(n connect.StreamingClientFunc) connect.StreamingClientFunc { return n }
func (i *AdminInterceptor) WrapStreamingHandler(n connect.StreamingHandlerFunc) connect.StreamingHandlerFunc {
    return func(ctx context.Context, conn connect.StreamingHandlerConn) error {
        uid, err := uuid.Parse(GetUserID(ctx))
        if err != nil { return connect.NewError(connect.CodeUnauthenticated, err) }
        ok, _ := i.checker.IsAdmin(ctx, uid)
        if !ok { return connect.NewError(connect.CodePermissionDenied, fmt.Errorf("admin required")) }
        return n(ctx, conn)
    }
}
```

2. `service.PlatformService.IsAdmin(ctx, userID)` 已存在（`@/opt/ant/backend/internal/service/platform_service.go:245-249`），直接用作 `AdminChecker`，但要把签名从 `string` 改成 `uuid.UUID`。

3. `backend/cmd/server/main.go:139` 后加：

4. `main.go:340-365` 把 7 个 `NewAdminXxxServiceHandler` 的拦截器从 `authInterceptor` 改为 `authInterceptor, adminInterceptor`：

### S0.6 三层 idempotency PG 锁修或删

**问题**：V2 §1.S-5。`@/opt/ant/backend/internal/mthub/idempotency.go:102-117` session-level 锁在不同连接释放。

**修复**：改 transaction-scoped：
```go
func (g *ThreeLayerGuard) tryAdvisoryLock(ctx context.Context, accountID, clientID string) (acquired bool, unlock func(), err error) {
    k1, k2 := advisoryLockKey(accountID, clientID)
    tx, err := g.pg.Begin(ctx)
    if err != nil { return false, nil, err }
    var ok bool
    err = tx.QueryRow(ctx, "SELECT pg_try_advisory_xact_lock($1, $2)", k1, k2).Scan(&ok)
    if err != nil { tx.Rollback(ctx); return false, nil, err }
    if !ok { tx.Rollback(ctx); return false, nil, nil }
    unlock = func() {
        // pg_advisory_xact_lock 在 commit/rollback 时自动释放
        _ = tx.Commit(ctx)
    }
    return true, unlock, nil
}
```

**或者**（如果短期不准备启用三层）：把整个 `ThreeLayerGuard` 移到 `legacy/` 目录或加注释 `// DEPRECATED: not wired, see TODO M11-3 v2`。
