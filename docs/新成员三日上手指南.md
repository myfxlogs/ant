# 新成员三日上手指南

> **与「一站式导读」的关系**：若只需**尽快全局了解**，请先读 **[新成员一站式导读](./新成员一站式导读.md)**（约 45～90 分钟）；**本文以下为选修**，用于按日拆任务、代码纵切与入职考核。  
> **目标**（选修路径）：在 **3 个工作日内**建立对 AntTrader 的**可操作认知**，并能从 **`docs/文档索引与维护状态.md`** 自主导航到正确文档。  
> **前提**：具备 Go / TypeScript / Python 之一熟练、其余愿意边查边学。

---

## 开始前（0.5 小时）

1. 克隆仓库，配置 IDE，确认能阅读 `docs/` 与主要目录 `frontend/`、`backend/`、`strategy-service/`、`proto`（或 buf 配置路径）。
2. 若尚未阅读：**[新成员一站式导读](./新成员一站式导读.md)** 全文；再读 **[文档索引与维护状态](./文档索引与维护状态.md)** 与 **[项目文档与进度管理体系](./项目文档与进度管理体系.md)** §1、§6、§7。

## 第二日：代码纵切

### 上午：选一条「读得懂」的链路

任选 **A 前端页面 → Connect 调用** 或 **B 后端 Connect 入口 → 内部服务**：

- 在 `frontend/src` 找一个业务页（如策略调度、市场状态），定位其使用的 `*Client` 与生成类型（`src/gen`）。
- 在 `backend` 找到对应 Connect 注册与 handler 文件。

### 下午：域文档与 proto

| 序号 | 任务 | 产出 |
|------|------|------|
| 1 | 在 `docs/domains/` 或 `docs/专项设计/` 找与所选链路相关的 1–2 篇 | 写 5 行摘要：设计意图与已知限制 |
| 2 | 在仓库中找到对应 `.proto` 变更入口（buf / Makefile 提示） | 知道「改接口要先改 proto 再 `make proto`」 |

### 日终自检

- 若你要加一个新 RPC，**最先**改哪两个地方（文档 + 代码层面）？
- 生成代码 `frontend/src/gen` 可以手改吗？为什么？
