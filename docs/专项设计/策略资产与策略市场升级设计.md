# 策略资产与策略市场升级设计

本文档定义 `strategy/assets` 后续升级为策略资产库与策略市场的产品边界、数据流、接口方向和实施阶段，作为后续开发依据。

## 1. 背景

当前 `strategy/assets` 已具备策略资产的雏形能力：

- 用户可将策略模板提交为资产。
- 资产具备可见性、审核状态、版本号、评分摘要和克隆次数等字段。
- 用户可将资产克隆为自己的策略模板草稿。

但当前能力仍偏基础，容易与策略模板的“公开/私有”功能产生重叠。

后续应将 `strategy/assets` 明确升级为：

```text
策略资产库 + 策略市场 + Agent 可检索策略素材库
```

其中：

- 策略资产库负责沉淀经过审核、评分、版本化的策略资产。
- 策略市场负责资产上架、购买、授权、收益结算和交易记录。
- Agent 可检索策略素材库负责为 AI 编写策略、优化策略和生成调度草案提供可复用资产。

## 2. 目标与非目标

目标：

- 明确 `StrategyTemplate`、`StrategyAsset`、`StrategyMarketListing` 的职责边界。
- 支持策略资产提交、审核、版本化、评分和克隆。
- 支持策略资产上架到策略市场。
- 支持用户购买、订阅、授权使用策略资产。
- 支持资产作者获得收益记录。
- 支持 Agent 检索适合当前行情和目标的策略资产。
- 所有业务能力统一通过 ConnectRPC 暴露。

非目标：

- 不允许购买资产后绕过模板校验、回测、调度前检查和 RiskEngine。
- 不允许策略资产直接变成交易指令。
- 不允许前端计算评分、风控结论、收益分成或交易结算。
- 不在第一阶段实现复杂二级市场、竞价系统或链上资产交易。
- 不把简单公开模板等同于可交易策略资产。

## 3. 核心概念边界

### 3.1 StrategyTemplate

`StrategyTemplate` 是用户工作区中的策略模板。

它表达：

- 草稿。
- 编辑中策略。
- 已发布模板。
- 可调度模板。
- 用户自己的策略源代码和参数。

特点：

- 可频繁修改。
- 与用户账户和调度流程强相关。
- 是策略开发和运行的基础对象。
- 可公开或私有，但公开不代表可交易。

### 3.2 StrategyAsset

`StrategyAsset` 是经过提交、审核、评分和版本化沉淀的策略资产。

它表达：

- 一个可复用策略包。
- 一个可被克隆的策略资产。
- 一个可进入市场上架流程的标的。
- 一个可被 Agent 检索的策略素材。

特点：

- 不应频繁直接修改。
- 应保留资产版本和快照。
- 应记录回测、评分、适用市场状态和风险等级。
- 克隆后生成用户自己的 `StrategyTemplate` 草稿。

### 3.3 StrategyMarketListing

`StrategyMarketListing` 是策略资产在策略市场中的上架条目。

它表达：

- 某个资产版本是否在市场售卖。
- 售卖价格或订阅价格。
- 授权方式。
- 购买规则。
- 市场展示信息。
- 上架状态。

特点：

- 一个资产可以不上架。
- 一个资产可以有多个版本。
- 一个资产版本可以对应一个市场 listing。
- 市场 listing 不改变资产本身内容。

## 4. 推荐对象关系

```text
StrategyTemplate
  -> SubmitAssetReview
  -> StrategyAsset
  -> StrategyAssetVersion
  -> StrategyMarketListing
  -> StrategyAssetPurchase / StrategyAssetLicense
  -> CloneStrategyAsset
  -> New StrategyTemplate Draft
```

实验候选也可以进入资产链路：

```text
StrategyExperimentCandidate
  -> PromoteCandidateToDraft
  -> Backtest / Score / Review
  -> SubmitAssetReview
  -> StrategyAsset
  -> StrategyMarketListing
```

Agent 使用资产链路：

```text
MarketAnalysisContext
  -> QuerySuitableAssets
  -> SelectAsset
  -> Clone / Adapt / Experiment
  -> Draft / ScheduleDraft
```

## 6. 策略市场核心流程

### 6.1 资产提交审核

```text
StrategyTemplate
  -> SubmitAssetReview
  -> 创建 StrategyAsset 草稿或待审核资产
  -> 生成 StrategyAssetVersion 快照
  -> 审核
  -> approved / rejected
```

要求：

- 提交时必须保存代码、参数和风控快照。
- 审核通过前不得上架市场。
- 审核拒绝必须保存原因。

### 6.2 资产上架市场

```text
Approved StrategyAssetVersion
  -> CreateMarketListing
  -> pending_review
  -> active
```

要求：

- 只能上架审核通过的资产版本。
- 上架文案、价格和授权条款需要审核。
- 上架后资产版本内容不可变。
- 如需修改策略内容，必须创建新版本并重新审核。

### 6.3 用户购买资产

```text
MarketListing
  -> CreatePurchase
  -> Payment / Internal Credit
  -> GrantLicense
  -> Buyer can clone asset
```

要求：

- 购买记录必须绑定具体资产版本。
- 授权记录必须明确有效期和使用限制。
- 未授权用户不得克隆付费资产。

### 6.4 克隆为策略模板

```text
StrategyAssetLicense
  -> CloneStrategyAsset
  -> New StrategyTemplate Draft
  -> ValidateTemplate
  -> Backtest / Score
  -> User Schedule Flow
```

要求：

- 克隆只能生成草稿模板。
- 不能购买后直接上线。
- 克隆后仍必须走模板校验、回测、调度和 RiskEngine。

### 6.5 资产版本升级

```text
Seller creates new asset version
  -> Review
  -> Publish version
  -> Notify licensed users
  -> Buyer chooses sync / keep current
```

要求：

- 用户克隆后的模板默认不被强制覆盖。
- 同步新版本前必须展示差异。
- 同步后仍需重新校验和回测。

## 8. 策略市场页面建议

### 8.1 `/strategy/assets`

定位：资产管理页。

用于：

- 查看我的资产。
- 提交资产审核。
- 管理资产版本。
- 查看审核状态。
- 查看克隆和购买统计。
- 进入上架流程。

### 8.2 `/strategy/market`

定位：策略市场页。

用于：

- 浏览公开上架资产。
- 搜索策略资产。
- 按策略类型、市场状态、评分、风险等级筛选。
- 查看资产详情。

- 克隆已授权资产。

### 8.3 `/strategy/market/:listingId`

定位：市场资产详情页。

建议展示：

- 策略介绍。

- 适用市场状态。

- 回测摘要。

## 9. 公开/私有与市场上架的关系

建议明确三层状态：

```text
private template
public template
market asset
```

区别：

| 能力 | 私有模板 | 公开模板 | 市场资产 |
|------|----------|----------|----------|
| 个人编辑 | 是 | 是 | 否，需版本化 |
| 他人可见 | 否 | 是 | 是 |
| 他人可复制 | 否 | 可选 | 购买或授权后可克隆 |
| 审核要求 | 无 | 可选 | 必须 |
| 评分沉淀 | 可选 | 可选 | 必须 |
| 版本快照 | 可选 | 可选 | 必须 |
| 收费交易 | 否 | 否 | 是 |
| Agent 检索 | 可选 | 可选 | 推荐支持 |

因此，公开模板不是策略市场资产。只有通过审核、版本化和上架流程的策略资产，才可以进入市场交易。

## 10. 与 Market Regime 和 Agent 的关系

策略市场不应只是模板商店，而应支持根据行情环境推荐资产。

推荐关系：

```text
MarketAnalysisContext
  -> regime = range
  -> QuerySuitableAssets(range)
  -> 推荐均值回归类策略资产
```

Agent 可以使用策略市场数据：

- 查询适合当前市场状态的资产。
- 解释某个资产为什么适合当前行情。
- 基于已授权资产生成策略草稿。
- 基于资产生成实验计划。
- 基于资产和候选生成调度草案。

限制：

- Agent 不能替用户购买资产。
- Agent 不能绕过授权克隆资产。
- Agent 不能直接启用使用资产的调度。

## 11. 风控与合规边界

策略市场交易的是策略资产或使用授权，不是收益承诺。

市场页面和资产详情必须避免：

- 承诺固定收益。
- 承诺无风险。
- 将历史回测等同未来表现。
- 用未经审核的收益数据做宣传。

必须展示：

```text
历史回测不代表未来收益。
策略资产购买后仍需用户自行配置、回测、验证和承担风险。
平台风控检查不构成投资建议。
```

技术边界：

- 买方购买后只能获得授权和克隆能力。
- 克隆后的模板仍需校验和回测。
- 正式调度仍需 RiskEngine 放行。
- 平台应能暂停高风险或违规资产。

## 12. 权限模型

建议权限：

| 权限 | 说明 |
|------|------|
| `asset:read` | 读取资产 |
| `asset:write` | 创建和编辑自己的资产 |
| `asset:review` | 审核资产 |
| `asset:clone` | 克隆有权限资产 |
| `market:read` | 浏览市场 |
| `market:write` | 创建上架 |
| `market:review` | 审核上架 |
| `market:purchase` | 购买资产 |
| `market:settle` | 查看和处理结算 |
| `agent:asset_read` | Agent 读取资产摘要 |

## 13. 分阶段实施建议

### 阶段 1：资产库补强

- 增加资产版本表。
- 提交资产时保存代码、参数、风控和评分快照。
- 完善审核状态和审核意见。
- 增加适用市场状态、风险等级和评分摘要。

### 阶段 2：市场上架基础能力

- 新增市场 listing 表。
- 新增 `StrategyMarketService`。
- 支持资产版本上架、暂停、下架。
- 前端新增 `/strategy/market` 浏览页。

### 阶段 3：购买与授权

- 新增购买记录和授权表。
- 支持免费、一次性购买和订阅授权。
- 克隆资产前检查授权。
- 新增我的购买和我的授权页面。

### 阶段 4：收益与结算

- 新增结算记录。
- 记录平台费用和作者净收入。
- 支持销售统计和收益查询。
- 后续再接入真实支付或内部积分系统。

### 阶段 5：Agent 检索与推荐

- 新增 `AgentStrategyAssetService`。
- 支持基于 MarketAnalysisContext 推荐资产。
- 支持解释资产适配原因。
- 支持基于已授权资产生成草稿和实验计划。

### 阶段 6：版本同步与生态治理

- 支持资产版本升级通知。
- 支持授权用户选择同步。
- 支持举报、风控下架和违规处理。
- 支持资产质量榜单和推荐排序。

## 14. 验收标准

完成后应满足：

- `strategy/assets` 明确作为资产管理页，不再只是公开模板列表。
- 策略资产具备版本快照、评分摘要、风险等级和适用市场状态。
- 只有审核通过的资产版本可上架市场。
- 用户可在策略市场浏览、筛选、购买或订阅资产。
- 购买后生成授权记录，授权决定是否可克隆。
- 克隆后只生成策略模板草稿，不直接上线。
- 调度仍必须经过模板校验、回测、调度前检查和 RiskEngine。
- Agent 可以检索适合当前行情的资产，但不能绕过购买、授权和风控。
- 历史购买、授权、克隆和结算可追溯。
- 前端不承担交易结算、评分、风控或授权判定。
