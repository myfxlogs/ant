# 策略资产库专项设计

本文档定义 AntTrader 借鉴 QuantDinger 策略共享与策略市场能力时的策略资产库、审核、克隆、版本、同步、公开分发、交易授权与收益结算边界。

## 1. 目标与非目标

目标：

- 将稳定策略模板沉淀为可复用、可审核、可版本化的策略资产。
- 支持团队内、跨租户和公开市场的策略资产发布、审核、克隆、版本记录和同步提示。
- 支持策略资产上架市场、免费获取、付费购买、积分兑换、会员授权和佣金分成。
- 支持资产购买、授权、收益结算、销售统计和授权到期管理。
- 为 Agent 和 MCP 提供受控的策略检索与复用能力。
- 避免发布者修改导致使用者调度行为突变。

非目标：

- 不在第一阶段实现复杂二级市场、竞价系统或链上资产交易。
- 不承诺策略资产收益，不把历史回测等同未来收益。
- 不允许 Agent 替用户自动购买资产或绕过授权。
- 不允许共享资产绕过策略校验、回测建议或调度启用检查。

## 2. 现有依赖模块

- StrategyTemplateService：策略模板草稿、发布与校验。
- BacktestRunService：策略资产质量证明与回测记录。
- CodeAssistService：AI 生成策略进入资产库前的校验链路。
- 审计日志：记录发布、审核、克隆、同步操作。
- Agent Gateway：Agent 读取或复用资产库的权限入口。
- 支付、积分或会员系统：策略资产交易、订阅和授权来源。
- 结算与审计服务：记录购买、分成、退款、授权和风控下架。

## 3. QuantDinger 可借鉴点

- 策略或指标发布、审核、克隆、本地副本同步。
- 使用者获取独立副本，源更新后提示同步。
- 社区资产的元数据、评分、复用次数和版本记录。
- 策略市场、付费授权、订阅、积分兑换、作者分成和公开分发。

AntTrader 可分阶段实施：先补齐内部资产库与版本快照，再升级为公开策略市场与交易授权体系。

## 4. AntTrader 实施边界

推荐定位：

```text
StrategyTemplate
  -> Review / Publish
  -> StrategyAsset
  -> StrategyAssetVersion
  -> StrategyMarketListing
  -> Purchase / License
  -> CloneToWorkspace
  -> Version / Sync Notice
  -> Validate / Backtest / Schedule
```

调度只能引用用户自己的模板副本，不直接引用共享源模板或市场资产源版本。

## 6. 状态与流程

发布流程：

```text
Draft Template
  -> Backend Validate
  -> Submit Review
  -> Review Approved
  -> SharedStrategyAsset
```

上架流程：

```text
Approved StrategyAssetVersion
  -> Create Market Listing
  -> Listing Review
  -> Active Market Listing
```

交易与授权流程：

```text
Active Market Listing
  -> Purchase / Points Exchange / Membership Check
  -> Grant License
  -> CloneToWorkspace
  -> Independent StrategyTemplate Draft
  -> Validate / Backtest / Schedule
```

克隆流程：

```text
SharedStrategyAsset
  -> CloneToWorkspace
  -> Independent StrategyTemplate
  -> Validate / Backtest
  -> Schedule
```

同步流程：

```text
Source Updated
  -> sync_available = true
  -> Show Diff Summary
  -> User Confirm
  -> Update Clone Draft
  -> Backend Validate
```

已启用调度的模板同步后必须重新经过后端校验，不得自动覆盖正在运行的调度行为。

市场资产上架后，资产版本内容不得原地修改。如需调整策略代码、参数 schema 或风控配置，必须创建新资产版本并重新审核。

## 7. ConnectRPC 接口草案

```text
StrategyAssetService
  ListStrategyAssets
  GetStrategyAsset
  SubmitAssetReview
  ReviewStrategyAsset
  CreateAssetVersion
  ListAssetVersions
  CloneStrategyAsset
  CheckAssetUpdate
  SyncStrategyAsset
  ListAssetClones
```

市场接口：

```text
StrategyMarketService
  CreateMarketListing
  UpdateMarketListing
  SubmitListingReview
  ReviewMarketListing
  ListMarketListings
  GetMarketListing
  PauseMarketListing
  RemoveMarketListing
```

交易与授权接口：

```text
StrategyAssetPurchaseService
  CreateAssetPurchase
  ConfirmAssetPurchase
  ListMyPurchases
  ListMySales
  GrantAssetLicense
  GetAssetLicense
  ListMyLicenses
  RevokeAssetLicense
```

结算接口：

```text
StrategyMarketSettlementService
  GetSellerBalance
  ListSettlementRecords
  RequestPayout
  GetPayoutStatus
```

权限建议：

- `ListStrategyAssets`、`GetStrategyAsset`：普通用户按可见性读取，Agent 需 `R` scope。
- `SubmitAssetReview`、`CloneStrategyAsset`、`SyncStrategyAsset`：需要写权限，Agent 需 `W` scope。
- `CreateAssetPurchase`、`GrantAssetLicense`：必须校验用户身份、支付或积分结果、会员状态和资产授权规则。
- 如果同步后触发回测或实验，还需要 `B` scope。
- 审核操作需管理员、团队管理员或市场审核权限。
- Agent 不得自动购买资产，不得绕过授权克隆付费资产。

## 8. 后端、策略服务与前端职责

后端负责：

- 资产可见性、审核状态、克隆关系和版本同步。
- 市场上架、购买、授权、结算、退款和下架状态。
- 发布、克隆、同步前后的模板校验编排。
- 审计日志、权限判断和跨租户访问控制。

策略服务负责：

- 策略代码校验。
- 回测与评分计算。

前端负责：

- 展示资产列表、审核状态、克隆状态、同步提示。
- 展示市场列表、资产详情、价格、授权状态、购买记录和收益统计。
- 展示 diff 摘要、风险提示和回测建议。
- 不计算策略评分、授权结论、结算结果或校验结论。

## 9. Agent / MCP 边界

Agent 可读策略资产库，但写操作必须受 scope 限制。建议工具：

```text
list_strategy_assets
get_strategy_asset
clone_strategy_asset
submit_asset_review
check_asset_update
sync_strategy_asset
query_market_assets
get_market_listing
recommend_asset_for_market_context
```

其中 `clone`、`submit_asset_review`、`sync` 属于写能力。

Agent 可基于 Market Regime、实验结果和用户目标推荐市场资产，但必须遵守：

- 不能替用户购买资产。
- 不能绕过授权克隆付费资产。
- 不能直接启用使用该资产的调度。
- 只能生成策略草稿、实验计划或调度草案。

## 10. 分阶段实施

1. 资产库数据模型、版本快照与只读列表。
2. 发布、审核、评分摘要和风险等级。
3. 克隆到个人工作区与源版本同步提示。
4. 市场上架、市场列表和资产详情页。
5. 免费获取、付费购买、积分兑换和会员授权。
6. 购买记录、授权记录、销售统计和佣金结算。
7. Agent/MCP 市场资产检索与行情适配推荐。
8. 写类工具在审计、授权和 scope 完备后开放。

## 11. 验收标准与禁止事项

验收标准：

- 发布资产必须来自通过后端校验的模板。
- 市场资产必须来自审核通过的资产版本。
- 资产版本必须保存代码、参数、风控、回测和评分快照。
- 用户可在市场中浏览、筛选、购买、积分兑换或通过会员权益获取策略资产授权。
- 购买或会员权益必须生成授权记录。
- 克隆后生成独立模板副本。
- 源更新不会自动改变使用者调度行为。
- 同步前有 diff 摘要和风险提示。
- 作者销售、佣金和结算记录可追溯。

禁止事项：

- 禁止共享资产绕过策略校验。
- 禁止未授权用户克隆付费或会员资产。
- 禁止把策略市场资产宣传为收益承诺。
- 禁止调度直接引用共享源模板。
- 禁止前端计算审核结论、策略评分、授权结论或结算结果。
