# 进度登记说明

## 1. 登记簿位置

- **机器可读**：[`模块进度登记簿.yaml`](./模块进度登记簿.yaml)  
- **体系说明**：[项目文档与进度管理体系](../项目文档与进度管理体系.md) §3

## 2. 何时更新

- 模块里程碑达成、范围裁剪、负责人变更、阻塞解除/出现时；
- 与合并默认分支的 PR **同一 PR** 内更新 YAML（便于 `git log` 形成时间线）；
- 百分比（`percent`）变化建议在 `summary` 中用一句话说明依据。

## 3. 字段约定（`模块进度登记簿.yaml`）

| 字段 | 必填 | 说明 |
|------|------|------|
| `module_id` | 是 | 稳定英文 id，用于脚本与后续 RPC |
| `name` | 是 | 中文展示名 |
| `owner` | 否 | 主要负责人，可与 Git 作者不同 |
| `status` | 是 | `not_started` / `in_progress` / `blocked` / `delivered`（可扩展） |
| `percent` | 是 | 0–100，对应当前里程碑共识；**不是**「全仓库所有设计文档目标的加权完成率」（全局定性见 **`docs/新成员一站式导读.md`** §2～§3） |
| `summary` | 是 | 一两句现状 |
| `blockers` | 否 | 阻塞项列表 |
| `doc_refs` | 否 | 相对 `docs/` 的路径或 URL |
| `updated_at` | 建议 | ISO 日期 `YYYY-MM-DD` |

## 4. 查询示例

- **按模块**：直接编辑/搜索 YAML 中 `module_id`。  
- **按时间线**：`git log --oneline -- docs/进度/`  
- **按人员**：搜索 `owner` 字段 + `git shortlog -sn -- docs/进度/`
